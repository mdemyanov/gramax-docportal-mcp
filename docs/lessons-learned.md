# Lessons Learned

Append-only журнал уроков команды агентов проекта `gramax-docportal-mcp`.

## Формат записи

Каждая запись — один пункт списка:

```
- YYYY-MM-DD — [роль/контекст] — урок и что из него следует
```

Примеры:

- 2026-05-08 — pm/процесс — фичи без research-фазы могут спотыкаться о незнакомые поля Gramax API; включай RES-NNN при триггерах «новый эндпоинт», «новая версия fastmcp».
- 2026-05-08 — dev/тесты — `respx` лучше httpx_mock для async fastmcp: даёт точный matcher по URL и body.

## Когда писать

- Открыли неочевидное требование процесса / методологии
- Нашли способ ускорить TDD-итерации
- Заметили систематическую ошибку в декомпозиции

Если урок применим только к одному файлу/функции — это комментарий в коде или commit message, не lessons-learned.

## Как использовать

PM при декомпозиции новой фичи бегло просматривает свежие записи (последние ~10), чтобы не повторять прошлых ошибок.

## Записи

- 2026-05-08 — pm/процесс — проджект-субагенты из `.claude/agents/` подхватываются Claude Code только при старте сессии. После их добавления нужен перезапуск CC, иначе Task tool отвечает «Agent type not found». Это smoke-проверено в сессии создания файлов.
- 2026-05-08 — pm/ревью — spec-compliance reviewer без знания структуры плана может флагать out-of-scope items как gap (Task 2 ai_search: ревьюер потребовал тесты edge-cases, которые отнесены к Task 3). Решение: в промпте ревьюера всегда явно описывать scope текущей таски и что отнесено к следующим — иначе цикл re-review бесконечен.
- 2026-05-08 — dev/worktree — `EnterWorktree` по умолчанию ветвится с `origin/<default>` (baseRef=fresh), а не с локального main. Если в локальном main есть незапушенные коммиты (план/спека/фикстура), они отсутствуют в новом worktree и тесты, ссылающиеся на эти артефакты, упадут. Фикс: сразу после входа в worktree сделать `git merge --ff-only main`.
- 2026-05-08 — dev/линтер — ruff `ASYNC109` запрещает параметр `timeout` в `async def`, рекомендуя `asyncio.timeout`. Для streaming-методов поверх `httpx.stream` это неприменимо: `asyncio.timeout` обернёт весь `async with`, а нам нужен per-request timeout, проброшенный в httpx. Используй `# noqa: ASYNC109` с комментарием-обоснованием на параметре, не отключай правило глобально.
- 2026-05-08 — dev/тесты — для мока `AsyncMock(spec=GramaxClient).ai_search` (метод-async-generator) присвоение `mock.ai_search = AsyncMock(...)` ломается: AsyncMock возвращает coroutine, не async-iterator. Нужно подменить атрибут на реальную async-gen-функцию: `async def fake(*a, **kw): yield ...; mock.ai_search = fake`. Для assert-call-count использовать holder-словарь со счётчиком внутри функции.
- 2026-05-08 — dev/парсер — Unicode-разделители в маркерах внешних API (например, CIT-маркеры Gramax с ZWSP+WJ) надо извлекать из реальной фикстуры, не из спецификации. Запиши декодированный sample в `tests/fixtures/` и проверяй парсер против него — гипотезы о codepoint'ах часто промахиваются.
