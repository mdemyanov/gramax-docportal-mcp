# ADR-0002: Опциональный API-токен — анонимный режим для публичных порталов

**Status:** Accepted
**Date:** 2026-06-10
**Author:** sa-agent

## Context

Gramax Doc Portal MCP изначально объявлял `GRAMAX_API_TOKEN` обязательной переменной
окружения (`gramax_api_token: str` в pydantic-settings).

Реальный портал https://knowledge.nau.im — публичный: все пять эндпоинтов API
отвечают 200 без заголовка `Authorization`. При этом:

1. **Crash при пустом токене (BUG-4).** Пользователи, у которых нет токена, задавали
   `GRAMAX_API_TOKEN=""`. Pydantic принимал это значение (тип `str`), клиент формировал
   заголовок `Bearer ` (с пробелом в конце) — httpx бросал `LocalProtocolError` до отправки
   запроса. Исключение не перехватывалось `GramaxError`, утекало к MCP-клиенту как
   необработанный traceback.

2. **Токен получается вручную через браузерную сессию** (`GET /api/user/token`) и может
   устаревать. Автообновления нет. Обязательность токена создавала ложный барьер входа.

3. **Невалидный токен → 401 на любом эндпоинте**, включая публичные. Это значит, что
   задать несуществующий токен «чтобы pydantic не ругался» — тоже не вариант.

Нужно было выбрать: жёстко валидировать непустой токен (вариант A) или сделать его
опциональным с анонимным fallback (вариант B).

## Decision

Принимаем **вариант B: токен опционален**.

- `config.py`: тип `str | None = None`; `@field_validator` нормализует пустую
  строку и whitespace к `None`, непустую строку — strip.
- `client.py`: заголовок `Authorization: Bearer <token>` добавляется только если
  `api_token` непустой; при `None` или `""` — заголовок не отправляется.
- Поведение с непустым токеном не меняется — PATCH-совместимость.

## Consequences

**Positive:**
- Пользователи публичных порталов (без токена) могут использовать сервер «из коробки»,
  без обходных мер — барьер входа снижен.
- Crash `LocalProtocolError` при `GRAMAX_API_TOKEN=""` устранён полностью.
- Конфигурация честно отражает реальность: токен нужен только для закрытых порталов
  или персонализированных операций.

**Negative:**
- Пользователь может случайно запустить сервер без токена против защищённого портала
  и получить 401 только при первом реальном запросе, а не при старте.
- Ошибки конфигурации (опечатка в токене) молчат на старте — диагностируются по 401.

**Mitigations:**
- 401 уже корректно обрабатывается через BUG-3 HTTP-error-handling: пользователь
  получает русское сообщение с HTTP-кодом, а не traceback.
- README обновляется: явно указано, что без токена работает анонимный режим, и когда
  токен нужен.
- Whitespace-нормализация (`strip`) предотвращает случайные ошибки с незримыми
  пробелами в `.env` файлах.

## Alternatives Considered

- **Вариант A: жёсткая валидация непустого токена** — отклонён. Решает crash при
  `""`, но не устраняет барьер для пользователей публичных порталов. Требует задавать
  фиктивный токен — что само по себе опасно (фиктивный токен даёт 401).
- **Вариант C: отдельный флаг `GRAMAX_ANONYMOUS=true`** — отклонён. Два независимых
  поля (`api_token` + `anonymous`) создают конфликтующие состояния (`token != None`
  и `anonymous=true` одновременно). Вариант B выражает то же семантически через
  `token is None`, без дополнительной переменной.
